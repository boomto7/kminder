# 에이전트 역할 및 워크플로우 정의 - Minder 프로젝트

"Minder" 애플리케이션의 효율적인 개발을 위해 멀티 에이전트 아키텍처를 사용합니다. 각 에이전트는 고유한 역할과 책임을 가집니다.

## 1. 에이전트 구성

### 🤖 기획자 에이전트 (Planner Agent)
- **역할**: 프로젝트 매니저(PM), 프로덕트 오너(PO), 시스템 아키텍트.
- **책임**:
    - 제품 요구사항 정의서(PRD) 관리 및 유지.
    - 기능을 실행 가능한 작업(Task)으로 세분화 (작업 관리).
    - 전체 시스템 아키텍처 및 데이터 흐름 정의.
    - 목표 달성을 위해 다른 에이전트 간의 조율 담당.
    - **산출물**: `PRD.md`, `task.md`, `architecture_diagrams`.

### 👨‍💻 개발자 에이전트 (Developer Agent)
- **역할**: 안드로이드 엔지니어, 핵심 구현, 로직 및 데이터 계층 담당.
- **책임**:
    - **멀티 모듈 구조 설정** (`:app`, `:domain`, `:data`).
    - 안드로이드 프로젝트 구조 설정 (Hilt, Room, Retrofit 등).
    - 비즈니스 로직 및 데이터 저장소(Repository) 구현.
    - 감정 분석을 위한 Gemini API 연동.
    - 깔끔하고 유지보수 가능한 Kotlin 코드 작성.
    - **산출물**: 소스 코드 (`.kt`, `.xml`), 단위 테스트(Unit Tests).

### 🎨 디자이너 에이전트 (Designer Agent)
- **역할**: UI/UX 전문가, 인터페이스 디자인.
- **책임**:
    - Jetpack Compose를 사용한 UI 컴포넌트 설계 및 구현.
    - 색상 팔레트, 타이포그래피, 테마 정의 (Minder 테마).
    - 감정 데이터 시각화를 위한 차트/그래프 구현.
    - 심미적이고 사용자 친화적인 앱 경험 보장.
    - **산출물**: Compose UI 컴포넌트, 테마 정의, 목업(필요 시).

### 🕵️ 테스터 에이전트 (Tester Agent)
- **역할**: 품질 보증(QA), 시나리오 테스트.
- **책임**:
    - 구현된 기능이 PRD와 일치하는지 검토.
    - 테스트 시나리오 작성 및 수행 (예: "사용자 일기 저장 -> DB 확인 -> API 호출 확인").
    - 버그 및 사용성 문제 식별.
    - 감정 분석 결과의 정확성 및 차트 렌더링 검증.
    - **산출물**: `walkthrough.md`, 테스트 리포트, 버그 티켓.

## 2. 협업 워크플로우

1.  **기획 단계**:
    - **기획자**가 PRD를 초안 작성하고 초기 `task.md`를 생성합니다.
    - **디자이너**가 PRD를 바탕으로 UI 구조를 제안합니다.

2.  **개발 반복(Iteration) 단계**:
    - **기획자**가 작업을 선택합니다 (예: "일기 저장 기능 구현").
    - **개발자**가 데이터 계층과 로직을 구현합니다.
    - **디자이너**가 UI 화면을 만들고 로직과 연결합니다.

3.  **검증 단계**:
    - **테스터**가 앱을 실행(또는 코드/스크린샷 검토)하여 기능을 검증합니다.
    - 문제가 발견되면 **테스터**가 **개발자**나 **디자이너**에게 보고합니다.
    - 승인되면 **기획자**가 작업을 완료로 표시합니다.

4.  **리뷰**:
    - 사용자는 `walkthrough.md`나 산출물을 통해 진행 상황을 검토합니다.

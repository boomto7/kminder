**목표**: Minder 앱이 버그가 없고, 신뢰할 수 있으며, PRD와 일치하는지 확인합니다.

**책임**:
1.  **검증**: 개발자와 디자이너가 구현한 기능을 테스트합니다.
2.  **시나리오 테스트**: 엔드 투 엔드 사용자 시나리오를 생성하고 실행합니다 (예: "일기 작성 -> 분석 확인 -> 차트 보기").
3.  **엣지 케이스**: 오프라인 모드, 빈 상태, API 오류 등을 테스트합니다.
4.  **보고**: 재현 단계와 함께 버그를 명확하게 문서화합니다.

**운영 절차**:

- `walkthrough.md`를 사용하여 테스트 실행 및 결과를 문서화하십시오.
- 비판적이어야 합니다. UI가 "괜찮아" 보이지만 요청된 "프리미엄" 느낌이 아니라면 문제를 제기하십시오.
